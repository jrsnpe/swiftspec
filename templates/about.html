Informações para Usuários e Desenvolvedores do SwiftSpec

Sobre o SwiftSpec

SwiftSpec é uma ferramenta revolucionária de documentação de API, projetada para transformar o processo de criação e manutenção de especificações de API. Com foco na eficiência e clareza, SwiftSpec permite que desenvolvedores e equipes técnicas criem documentação de API de alta qualidade em uma fração do tempo tradicional.

Geração Rápida

Crie documentação completa em questão de segundos, economizando tempo valioso.

Clareza Incomparável

Transforme códigos complexos em documentação cristalina e fácil de entender.

Design Impressionante

Crie APIs que não só funcionam bem, mas também impressionam visualmente.

Instalação

Para instalar o SwiftSpec via npm, execute o seguinte comando:

npm install -g swiftspec

Uso Básico

Após a instalação, você pode usar o SwiftSpec da seguinte maneira:

swiftspec generate

Iniciar o servidor para visualizar a documentação já existente:

swiftspec serve

Gerar a documentação e iniciar o servidor em um único comando:

swiftspec start

Utilização Manual

Gerar documentação para um projeto específico:

swiftspec generate ./meu-diretorio-origem ./meu-diretorio-saida

Iniciar o servidor em uma porta específica:

swiftspec serve ./meu-diretorio-saida 8080

Gerar e servir documentação em uma porta específica:

swiftspec start ./meu-diretorio-saida 8080

Configuração

Configure o SwiftSpec criando um arquivo swiftspec.config.js na raiz do seu projeto:

module.exports = {
  title: 'TaskMaster API',
  version: '2.1.3',
  description: 'API RESTful para o sistema de gerenciamento de tarefas TaskMaster',
  baseUrl: 'https://api.taskmaster.com/v2',
  // Mais opções de configuração...
};

Exemplos

Aqui está um exemplo de como documentar um endpoint em seu código:

/**
 * @api {get} /tasks/:id Obter tarefa por ID
 * @apiName GetTask
 * @apiGroup Tasks
 * @apiParam {Number} id ID único da tarefa
 * @apiSuccess {String} title Título da tarefa
 * @apiSuccess {String} description Descrição da tarefa
 * @apiSuccess {Boolean} completed Status de conclusão da tarefa
 */
app.get('/tasks/:id', (req, res) => {
  // Lógica para obter tarefa
});

Para Desenvolvedores

Baixar do GitHub

Para clonar o repositório do SwiftSpec, use o seguinte comando:

git clone https://github.com/swiftspec/swiftspec.git

Setup do Projeto

  1. Certifique-se de ter o Node.js instalado (versão 14 ou superior)
  2. Navegue até o diretório do projeto: 
    cd swiftspec
  3. Instale as dependências: 
    npm install
  4. Para iniciar o servidor de desenvolvimento: 
    npm run dev

Configuração

Se a configuração não iniciar automaticamente, você pode executá-la manualmente:

npx swiftspec configure

Se a configuração ficou incorreta, você pode remover e iniciar o processo novamente:

npx swiftspec unconfigure

Para exclusão manual do arquivo de configuração:

  • Localize e exclua o arquivo swiftspec.config.json na raiz do seu projeto.
  • Após remover a configuração, o SwiftSpec usará as configurações padrão em futuras execuções, ou você pode reconfigurar.

Como Contribuir

  1. Faça um fork do repositório
  2. Crie uma branch para sua feature: git checkout -b feature/nova-funcionalidade
  3. Faça commit das suas alterações: git commit -am 'Adiciona nova funcionalidade'
  4. Faça push para a branch: git push origin feature/nova-funcionalidade
  5. Crie um novo Pull Request

Diretrizes de Código

  • Siga o estilo de código estabelecido no projeto (utilizamos ESLint e Prettier)
  • Escreva testes unitários para novas funcionalidades
  • Mantenha a documentação atualizada, incluindo JSDoc nos componentes principais
  • Use nomes descritivos para variáveis e funções
  • Priorize código legível e auto-explicativo, usando comentários apenas quando necessário

Executando Testes

Para executar os testes do projeto:

npm test

Gerando a Documentação

Para gerar a documentação do projeto SwiftSpec:

npm run generate-docs

A documentação será gerada no diretório docs/
